RDBMS의 표준이자 금융권에서 가장 많이 사용되는 오라클 데이터베이스를 공부하기 위해 `초보자를 위한 Oracle 12c` 책의 각 챕터를 읽고난 후 중요 내용을 정리해보기로 하였다.

# DBMS의 목적

Database Management System

최초 DBMS : 파일처리 시스템의 문제점을 해결하기 위해 등장

```bash
문제점 : 서로 다른 형식의 파일과 다양한 응용프로그램 등장
-> 동일한 정보가 여러 파일에 중복되어 저장됨
-> 특정 하나의 파일에서 값 변경하면, 서로 다른 값을 가지게 되어 `데이터 일관성` 에 큰 문제 발생
```

이러한 `정합성의 문제점`  을 해결 + 좀 더 효율적으로 데이터를 `검색` 하기 위해 DBMS 등장

결국, DBMS의 궁극적인 목표

- 물리적 독립성
- 접근의 독립성
- 데이터의 독립성

# DBMS 특징
 
> 정보시스템 개발을 컴퓨터가 아닌, 인간이 작업하는 방식이랑 유사하도록!

- 데이터의 일관성 + 정합성 보장 

- 백업 및 복구 기능

- 질의 (Query) 처리

# 관계형 DBMS (RDBMS)

> 관계 데이터 모델 → 테이블 기반

- 상용 RDBMS : 오라클, MS SQL, 사이베이스, DB2, 버티카

- 오픈 RDBMS : MySQL, 마리아 DB
